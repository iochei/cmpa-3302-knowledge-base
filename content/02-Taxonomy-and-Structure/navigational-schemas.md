Navigational schemas are the systems that allow users to move through the information architecture you have created. While [Organization Systems](02-Taxonomy-and-Structure/organization-systems.md) define how content is grouped, navigation systems define the paths the user takes.

In IA, we categorize navigation into three primary types:

### 1. Global Navigation
*   **Definition:** The main navigation menu that appears on every page of the site.
*   **Purpose:** Provides access to the top-level sections of the site. It orients the user and answers the question, "Where else can I go?" (e.g., main site header, footer links).

### 2. Local Navigation
*   **Definition:** Navigation specific to the current section of the site you are in.
*   **Purpose:** Allows users to explore the sub-pages within a category. It answers the question, "Where can I go in this section?" (e.g., a sidebar menu on a sub-page).

### 3. Contextual Navigation
*   **Definition:** Hyperlinks embedded directly within the content body.
*   **Purpose:** Connects the user to related information, regardless of where it lives in the hierarchy. It answers the question, "What is related to this specific concept?" (e.g., the links in the "Related Concepts" section at the bottom of this page).

### The Hierarchy of Navigation

Effective IA uses all three. Global navigation is **top-down**, providing broad access. Contextual navigation is **bottom-up**, providing granular, semantic connections.
