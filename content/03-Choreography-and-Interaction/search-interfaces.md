While browsing and navigation rely on [Organization Systems](02-Taxonomy-and-Structure/organization-systems.md) and [Hierarchies](02-Taxonomy-and-Structure/hierarchical-structures.md), search systems rely on direct query. An effective Information Architecture must integrate both. The search system interface is the primary way users interact with this retrieval mechanism.

### Anatomy of an Effective Search Interface

Designing the search interface involves several IA considerations:

1.  **The Input Box:** Simple, standard, and prominent. It should be consistent across all pages (often in the global navigation).
2.  **Query Autocomplete/Suggest:** IA systems can use controlled vocabularies and popularity data to suggest search terms as the user types, guiding them toward correct terminology.
3.  **Search Results Page:** This is critical. It must clearly display:
    *   How many results were found for the query.
    *   The query the system *actually* searched for (including any spell-corrections).
    *   A ranked list of results, with relevant titles, snippets, and metadata (e.g., date, category).
4.  **Faceted Search Results:** The most powerful search interfaces allow users to refine their search results using [Faceted Classification](02-Taxonomy-and-Structure/faceted-classification.md) (e.g., filtering by date, author, or topic after performing a keyword search).

The search interface is not just a box; it's a conversation between the user and the information system.
